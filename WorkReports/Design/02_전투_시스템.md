# 전투 시스템

## 구현 상태: 기본 완성 (AI 고도화 미구현)

---

## 핵심 파일

| 파일 | 역할 |
|------|------|
| `Assets/Scripts/Logic/Manager/TurnManager.cs` | 턴 진행, 몬스터 AI, 공격 처리 |
| `Assets/Scripts/Logic/Manager/StageManager.cs` | 액터 관리, 타일 선택, 승패 판정 |
| `Assets/Scripts/Logic/Scene/StageScript.cs` | 스테이지 진입/종료 |

## 턴 구조

```
StartGame()
    └──▶ [Pawn 턴]
              플레이어 행동 (이동, 카드 사용)
              EndPawnTurn() 버튼 클릭
              └──▶ [Monster 턴] (자동)
                        Alert 판정
                        각 몬스터 행동 순서대로 실행
                        모든 몬스터 완료
                        └──▶ [Pawn 턴] (카드 1장 드로우)
```

---

## Pawn 턴

### 이동
- 타일 클릭 → `StageManager.SelectTile()`
- 선택된 Pawn → 이동 범위 하이라이트 (BFS)
- 빈 타일 클릭 → A* 경로탐색 → `Actor.MoveTo()`
- 이동 후 잔여 이동력 기록 (`TurnManager._remainingMovement`)
- 이동력 0이 되면 이동 불가

### 카드 사용
- `StagePage`에서 카드 드래그&드롭으로 사용
- 자세한 내용은 `03_카드_시스템.md` 참고

### 턴 종료
- EndTurn 버튼 → `TurnManager.EndPawnTurn()`
- `StageManager.Deselect()` 호출 → 선택 해제
- Monster 턴으로 전환

## Monster 턴

### Alert 상태 판정
- 어느 한 몬스터의 Sight 범위 내에 Pawn이 1명이라도 있으면 **전체 Alert**
- Alert 상태: 모든 몬스터가 Pawn을 추적
- 비Alert 상태: 모든 몬스터가 배회(Wander)

### 몬스터 행동 순서 (Alert 상태)
```
1. 가장 가까운 Pawn 탐색
2. 이동 전 공격 시도 (사정거리 내이면 공격 후 행동 종료)
3. 아니면 Pawn 방향으로 이동
4. 이동 후 공격 시도 (사정거리 내이면 공격)
```

### 몬스터 행동 순서 (비Alert 상태 - 배회)
```
이동력 최대 2칸 제한으로 랜덤 이동
```

### 공격 처리 (`ExecuteAttack`)
```
1. 공격 모션 (Attack01 애니메이션)
2. 피해 적용: target.TakeDamage(attackPower)
   - 실피해 = attackPower - target.Armor
   - Shield 우선 감소 → HP 감소
3. GetHit 모션
4. 사망 체크 → Die() 애니메이션
5. StageManager.CheckBattleResult()
```

## 승패 판정

- **파일**: `StageManager.CheckBattleResult()`
- **이벤트**: `StageManager.onBattleEnd` (StagePage가 구독)
- **승리 조건**: `_dicEnemyActor` 내 모든 Actor가 비활성(`!gameObject.activeInHierarchy`)
- **패배 조건**: `_dicUserActor` 내 모든 Actor가 비활성
- **호출 시점**:
  1. `TurnManager.ExecuteAttack()` — 사망 애니메이션 완료 후
  2. `StagePage.UseCard()` — 카드 데미지로 몬스터 사망 후

---

## 액터 관리 (StageManager)

| 딕셔너리 | 내용 |
|---------|------|
| `_dicUserActor` | 플레이어 Pawn (int ID → Actor) |
| `_dicEnemyActor` | 몬스터 (int ID → Actor) |

- Actor 사망: `Actor.Die()` → `gameObject.SetActive(false)` + `Slot.ClearEntity()`
- 생존 판단: `gameObject.activeInHierarchy` 기준
