## UIManager

- **패턴**: `MonoSingleton` + `[AssetPath("Prefabs/Manager/UIManager")]`
- **프리팹**: `Resources/Prefabs/Manager/UIManager.prefab` (Inspector에서 Canvas 연결)

### Canvas 구조
```
UIManager (GameObject)
├── _canvasPage    : PageView 전용 Canvas (하위 레이어)
├── _canvasPopup   : PopupView 전용 Canvas (상위 레이어)
└── _hudController : HudController (HUD 관리 — FloatingText 풀 등)
```

### 내부 자료구조
```csharp
Dictionary<Type, ViewCore> _dicPage    // PageView 인스턴스 캐시
Dictionary<Type, ViewCore> _dicPopup   // PopupView 인스턴스 캐시
Stack<PageView> _pageStack             // Page 전환 히스토리 (LIFO)
```

### 주요 메서드
```csharp
T OpenView<T>(ViewParam param = null)
    → 딕셔너리에 없으면 AssetPath로 프리팹 로드 후 Instantiate
    → 이미 열려있으면 기존 인스턴스 반환 (스택/hide 로직 미적용)
    → PageView: 현재 top 페이지가 DontHide=false이면 SetActive(false)
    → _pageStack에 Push → PreOpen() → OnOpened() 순서로 호출

CloseView(ViewCore view)
    → PreClose() → 딕셔너리/스택에서 제거 → DestroyImmediate → OnClosed()
    → PageView: 스택 Pop 후 이전 top 페이지 SetActive(true) 복원

CloseAllView()
    → 열린 모든 뷰 PreClose() 후 DestroyImmediate
    → _dicPage, _dicPopup, _pageStack 모두 Clear
    → _hudController.ClearAll() — 활성 FloatingText 정리
```

## Page 전환 동작

PageView는 스택 기반으로 전환된다. 새 Page가 열리면 이전 Page는 숨겨지고, 닫히면 이전 Page가 복원된다.

```
LobbyPage 열림
  └─ _pageStack: [LobbyPage]

ShopPage 열림
  └─ LobbyPage.SetActive(false)
  └─ _pageStack: [LobbyPage, ShopPage]

ShopPage 닫힘
  └─ ShopPage DestroyImmediate
  └─ LobbyPage.SetActive(true)
  └─ _pageStack: [LobbyPage]
```

### DontHide 옵션
```csharp
// PageView 프리팹 Inspector에서 설정
[SerializeField] private bool _dontHide = false;
public bool DontHide => _dontHide;
```
- `false` (기본값): 새 Page 오픈 시 이 페이지가 숨겨짐
- `true`: 새 Page가 열려도 이 페이지는 숨겨지지 않고 유지됨

## View 라이프사이클

```
OpenView<T>() 호출
    → PreOpen()    : 버튼 리스너 등록, 이벤트 구독, 데이터 세팅
    → OnOpened()   : 초기 표시 업데이트 (카드 목록 갱신 등)

CloseView() 호출
    → PreClose()   : 이벤트 구독 해제, 리스너 제거
    → OnClosed()   : 정리 작업
```

## ViewParam

뷰 간 데이터 전달:
```csharp
public abstract class ViewCore
{
    public abstract class ViewParam { }
    protected ViewParam param;
}

// 사용 예:
UIManager.Instance.OpenView<BattleResultPopup>(
    new BattleResultPopup.BattleResultParam { isWin = true }
);
```

## 각 뷰 상세

### TitlePage
- New Game → LobbyScene
- Load Game → LobbyScene
- Option → OptionPopup
- Exit → 게임 종료

### LobbyPage
- Start Battle → StageScene
- Deck Setting → DeckSettingPage 오픈
- Shop → ShopPage 오픈
- Recruit → RecruitPage 오픈
- Exit → TitleScene

### ShopPage
- ShopData Level 기반으로 최대 10개 카드 진열 (5×2 그리드)
- 구매 시 ShopItem hide, DeckManager._deckCardList에 DCard 추가
- LevelUp 버튼으로 ShopData 최고 레벨까지 단계 상승 가능
- Stage 진행 전까지 진열 목록 유지 (Refresh() 호출 시 갱신)
- Close → `Close()` → LobbyPage 복원
- **관련 파일**: `09_상점_시스템.md`

### DeckSettingPage
- Pawn/Card 탭 전환 버튼 구현
- **Pawn 탭** (`DeckSetting_Pawn`): 보유 Pawn 목록 표시, 드래그 or 우클릭으로 슬롯 편성
- **Card 탭** (`DeckSetting_Card`): 미구현
- Close → `Close()` → LobbyPage 복원
- **관련 파일**: `10_덱편성_시스템.md`

### RecruitPage
- Close → `Close()` → LobbyPage 복원

### PawnManagePage
- **관련 파일**: `Assets/Scripts/UI/PawnManage/`
- 좌측 스크롤 리스트(`Scroll_PawnList`): 보유 Pawn을 `PawnManageItem`으로 표시 (가로 스크롤)
- 우측 스크롤 리스트(`Scroll_EquipList`): 보유 장비를 `PawnManageEquipItem`으로 4열 그리드 표시 (세로 스크롤)
- 장비 슬롯 영역(`EquipSlot`): Weapon, Armor, Tool×2 총 4슬롯

#### 인터랙션
| 액션 | 결과 |
|------|------|
| `PawnManageItem` 좌클릭 | 해당 Pawn 선택, `_goSelected` 활성화, 슬롯 갱신 |
| `PawnManageEquipItem` 드래그 → 슬롯 릴리즈 | 슬롯 타입 일치 시 선택 Pawn에 장착 |
| `PawnManageEquipItem` 우클릭 | 해당 타입의 슬롯에 자동 장착 (빈 슬롯 우선) |
| `PawnManageEquipSlot` 우클릭 | 장착 해제 |

#### 핵심 클래스
| 클래스 | 역할 |
|--------|------|
| `PawnManagePage` | 페이지 조율 — 선택·드롭·자동장착·해제 |
| `PawnManageItem` | Pawn 아이템 — 클릭 선택, `OnSelected` 이벤트 |
| `PawnManageEquipItem` | 장비 아이템 — 드래그 + 우클릭 이벤트 |
| `PawnManageEquipSlot` | 장비 슬롯 — 상태 표시(`_goEmpty`/`_spawnIcon`), 우클릭 해제 |

- Close → `Close()` → LobbyPage 복원

### StagePage
- **구독 이벤트**: `DeckManager.onHandChanged`, `StageManager.onBattleEnd`, `StageManager.onSelectedChanged`, `TurnManager.onTurnChanged`, `TurnManager.onSharedActingPowerChanged`
- **글로벌 카드 UI** (`_goCardGroup`): `DeckManager.handCardList` → `BattleCard` 동적 생성
- **Pawn 카드 UI** (`_goPawnCardGroup`): 선택된 Pawn의 `PawnHandCards` → `BattleCard` 동적 생성
  - Pawn 선택 시 해당 Pawn의 `OnPawnHandChanged` 이벤트 구독 (선택 해제 시 구독 해제)
- **버튼**: Pause(PausePopup), EndTurn(TurnManager.EndPawnTurn)
- **전투 결과 수신**: `StageManager.onBattleEnd` → `BattleResultPopup` 열기

### BattleResultPopup
- `BattleResultParam` 필드: `isWin`, `goldReward`, `expReward`
- **공통**: 골드/경험치 텍스트 표시 (`_textRewardGold`, `_textRewardExp`)
- **승리 시**: Continue 버튼 표시, `_goRewardCard` 활성화 → 카드 보상 3장 세팅
  - `StageRewardData.GetByLevel(DifficultyLevel)` → `cardId`/`cardProb` 가중치 랜덤 3장 (중복 불가)
  - `BattleResultItem` × 3: `CardIcon`으로 카드 정보 표시, 클릭으로 1장 선택/해제
  - Continue 클릭 → 골드/경험치 적용, 선택 카드 `DeckManager.AddCard()`, LobbyScene
- **패배 시**: Exit 버튼 표시, `_goRewardCard` 비활성화
  - Exit 클릭 → TitleScene
- Title 텍스트: "승리" / "패배"

### PausePopup

- Exit → 게임 종료

### OptionPopup

## HUD 시스템

### HudController
- **파일**: `Assets/Scripts/UI/HUD/HudController.cs`
- **역할**: UIManager 하위에서 HUD 관련 기능 관리 (현재: FloatingText 오브젝트 풀)
- **초기화**: `UIManager.Awake()` → `_hudController.Init()`
- **외부 접근**: `UIManager.Instance.Hud`

### FloatingText (플로팅 텍스트)
- **파일**: `Assets/Scripts/UI/HUD/FloatingText.cs`
- **프리팹**: `Resources/Prefabs/UI/FloatingText.prefab`
- **상속**: `MonoBehaviourEx` + `[AssetPath("Prefabs/UI/FloatingText")]`
- **용도**: 전투 중 데미지/회복/버프 등 수치 변화를 액터 위에 떠오르는 텍스트로 표시
- **호출 경로**: `DPawn/DMonster.onFloatingText` 이벤트 → `Actor.OnFloatingText()` → `UIManager.Instance.Hud.ShowFloatingText()`

#### FloatingTextType

| 종류 | 색상 | 연출 | 트리거 |
|------|------|------|--------|
| Damage | 빨강 | 대각 포물선 + 바운스 | `TakeDamage()` (net > 0) |
| Heal | 초록 | 수직 상승 | `Heal()` |
| Shield | 하늘 | 수직 상승 | `AddShield()` |
| Buff | 주황 | 제자리 팝 | `ApplyBuff(Buff*)` |
| Debuff | 보라 | 제자리 팝 | `ApplyBuff(Debuff*)` |
| Block | 회색 | 제자리 팝 | `TakeDamage()` (net == 0) |

#### 겹침 방지
- `HudController`가 액터별 활성 텍스트 수를 추적
- 동일 액터에 연속 텍스트 발생 시 Y축 오프셋 누적 (0.4 단위)
- 텍스트 완료 시 자동으로 카운트 감소

#### 오브젝트 풀
- 초기 10개 생성, 부족 시 자동 확장
- `PrefabLoader.Load<FloatingText>()`로 프리팹 로드 (AssetPath 기반)
- 비활성화된 인스턴스를 `Queue<FloatingText>`로 관리

## 아이콘 클래스

### CardIcon
- **파일**: `Assets/Scripts/UI/Icon/CardIcon.cs`
- **프리팹**: `Resources/Prefabs/UI/Icon/CardIcon.prefab`
- **상속**: `IconBase`
- **기능**: 카드 데이터 **표시 전용** (상점/로비 등 비전투 화면에서 사용)

### BattleCard
- **파일**: `Assets/Scripts/UI/HUD/BattleCard.cs`
- **프리팹**: `Resources/Prefabs/UI/HUD/BattleCard.prefab`
- **상속**: `MonoBehaviourEx` (CardIcon과 독립)
- **기능**: 전투 전용 카드 UI — 표시 + 드래그&드롭 이벤트 발행
