## 데이터 구조

### ShopData (자동생성 Protobuf)
```
Id       int               — 행 고유 ID
Level    int               — 상점 레벨
CardId   RepeatedField<int> — 진열할 카드 ID 목록
ItemId   RepeatedField<int> — (미사용, 향후 아이템 확장용)
```

### ShopTable 조회 방식
```csharp
// Load() 시점에 레벨→ShopData 맵 구축
DataManager.Instance.Shop.GetByLevel(level)  // 레벨로 ShopData 조회
DataManager.Instance.Shop.GetMaxLevel()       // 최고 레벨 반환
DataManager.Instance.Shop.Get(id)             // ID로 ShopData 조회
```

## ShopPage 동작 흐름

```
OnOpened()
  └─ LobbyManager.ShopCards == null → LobbyManager.GenerateShopList()
                                     + RefreshItems()
  └─ RefreshGoods()   (골드 표시만 갱신)

LobbyManager.GenerateShopList()
  └─ ShopTable.GetByLevel(_shopLevel) → ShopData
  └─ CardId 리스트 → CardData 수집 (최대 10개)

RefreshItems()
  └─ 기존 ShopItem 파괴
  └─ AssetPath Reflection으로 ShopItem 프리팹 로드
  └─ LobbyManager.ShopCards 기반 5×2 그리드 배치
  └─ item.SetData(cardData), OnBuyItem 이벤트 등록
```

### 카드 그리드 배치 (5열 × 2행)
```csharp
int row = i / 5;
int col = i % 5;
anchoredPosition = new Vector2(-440f + 220f * col, -row * 300f);
```
> 실제 간격은 ShopItem 프리팹 크기에 맞게 조정 필요

## ShopItem 동작

```csharp
SetData(CardData data)
  → _cardData 저장
  → _textPrice.text = data.Price > 0 ? data.Price.ToString() : "FREE"

OnClickBuy()
  → new DCard(_cardData)
  → DeckManager.Instance.AddCard(card)
  → gameObject.SetActive(false)   // hide 처리
  → OnBuyItem?.Invoke(this, card)
```

## 상점 레벨 시스템

- 최초 레벨: `LobbyManager._shopLevel = 1`
- 최고 레벨: `ShopTable.GetMaxLevel()` (데이터 기반 동적 산출)
- LevelUp 버튼 → `LobbyManager.LevelUpShop()` → `GenerateShopList()` → `RefreshItems()`
- 최고 레벨 도달 시 LevelUp 버튼 동작 없음

---

## 진열 유지 정책

- `LobbyManager.ShopCards` 가 `null` 이 아니면 목록 재생성 건너뜀
- LobbyManager는 MonoSingleton(DontDestroyOnLoad)이므로 페이지 전환과 무관하게 유지
- `ShopPage.Refresh()` 공개 메서드: Stage 종료 후 외부에서 호출 → `LobbyManager.GenerateShopList()` 호출

```csharp
// 예: StageScript에서 호출
UIManager.Instance.OpenView<ShopPage>()?.Refresh();
```
