## Pawn 데이터 (DPawn)

### 스탯 목록
| 스탯 | 설명 |
|------|------|
| HP | 현재 체력 |
| Shield | 보호막 (피해 우선 흡수) |
| Armor | 방어력 (피해 감소) |
| Movement | 이동력 |
| Attack | 공격력 |
| Range | 사거리 (카드 사정거리에 합산) |
| Sight | 시야 범위 |
| ActingPower | 행동력 (카드 사용 비용 차감, 턴 시작 시 회복) |

### 피해 계산
```
실제 피해 = attackPower - Armor
Shield 우선 감소 → Shield 소진 후 HP 감소
```

### 스탯 합산 구조

스탯은 `PawnStats` 컴포넌트가 합산하여 관리한다:
```
PawnStats.Recalculate(PawnData, IReadOnlyList<DEquipment>)
    → 기본값 = PawnData.*
    → 장비 StatusType/StatusValue 쌍을 순회하여 보너스 누산
    → DPawn.UpdateStatus()가 Stats.* 값을 런타임 필드에 복사
```

런타임 변동(피해, 버프)은 DPawn이 직접 관리하며, `ResetStats()`시 Stats 기저값으로 복원.

### 초기화
```csharp
DPawn(int pawnId)
    → PawnData 로드
    → Stats.Recalculate(Data, _equips)  // 장비 없으면 기본값만
    → UpdateStatus()                     // 런타임 필드에 복사
```

### 이벤트
- `onStatsChanged` — HP/Shield 변경 시 발생 (HUD 업데이트용)

## Actor (인스턴스)

### 상속 구조
```
MonoBehaviour
    └── MonoBehaviourEx
            └── TileEntity      (Data, Slot 보유)
                    └── Actor   (애니메이션, 이동)
```

### 초기화
```csharp
actor.Init(DPawn pawn)    // Pawn으로 초기화
actor.Init(DMonster mon)  // Monster로 초기화
```
- HUD_Actor 생성 및 연결
- `data.onStatsChanged` 구독

### 이동 (MoveTo)
```
속도: MoveSpeed = 3f (상수)
경로: List<SquareTile> (SmoothPath 적용된 웨이포인트)
처리:
    1. 현재 Slot.ClearEntity()
    2. 각 웨이포인트 순서대로 선형 이동
    3. 도착 Slot.SetEntity(this)
    4. onComplete 콜백 호출
```

### 애니메이션 상태
| 상태 이름 | 트리거 | 설명 |
|---------|--------|------|
| Attack01 | - | 공격 모션 |
| GetHit | - | 피격 모션 |
| Die | - | 사망 모션 → SetActive(false) |

### 사망 처리
```
Die(Action onComplete)
    → Animator "Die" 상태 진입
    → 애니메이션 완료 대기
    → Slot.ClearEntity()
    → gameObject.SetActive(false)
    → onComplete 호출
```

## 장비 시스템 (DEquipment)

```csharp
public class DEquipment : DItem
{
    public int equipmentId;
    public EquipmentData Data;  // 이름, 슬롯타입, 아이콘 경로 등
}
```

`DPawn`에 `_equips` 목록이 있으며 장착/해제 메서드 구현 완료:
```csharp
public PawnStats Stats { get; } = new PawnStats();
public IReadOnlyList<DEquipment> Equips => _equips;

public void Equip(DEquipment equip)    // 추가 후 RecalculateStats() 호출
public void Unequip(DEquipment equip)  // 제거 후 RecalculateStats() 호출
```

장착/해제 시 `PawnStats.Recalculate()` → `UpdateStatus()` → `onStatsChanged` 순으로 자동 갱신.

`ItemManager`가 보유 장비 목록(`_equipList`)을 관리:
```csharp
public IReadOnlyList<DEquipment> Equips => _equipList;
public void AddEquip(DEquipment equip) => _equipList.Add(equip);
```

### 장착 상태 추적 (DEquipment)

```csharp
public long EquippedPawnId { get; private set; } = 0;  // 0 = 미장착
public bool IsEquipped => EquippedPawnId != 0;
public void SetEquippedPawn(long pawnInstanceId) => EquippedPawnId = pawnInstanceId;
public void ClearEquippedPawn() => EquippedPawnId = 0;
```

`DObject.InstanceId`를 활용하여 어느 Pawn에 장착됐는지 추적.
`PawnManagePage`의 장비 목록은 `IsEquipped == false`인 장비만 표시.

### 장비 슬롯 타입 (EquipSlotType)
| 값 | 이름 | 설명 |
|----|------|------|
| 1 | WeaponMain | 주무기 |
| 2 | WeaponSub | 보조무기 |
| 3 | Armor | 방어구 |
| 4 | Helmet | 헬멧 |
| 5 | Tool | 도구 |

### 장비 스탯 (StatusType)

`EquipmentData.StatusType` / `StatusValue`는 repeated 쌍으로 정의되며, `PawnStats`가 합산한다:

| StatusType | 매핑 스탯 |
|------------|-----------|
| StatAtk | Attack |
| StatDef | Armor |
| StatHp | Hp (최대 HP 증가) |
| StatShield | Shield |
| StatMovement | Movement |
| StatRange | Range |
| StatCardCap | MaxHandSize 보너스 |
| StatAccuracy | (예비, 미사용) |

### Pawn 개별 카드풀 (DPawn)

각 Pawn은 장착 장비의 `EquipmentData.CardId` 목록으로 자신만의 카드풀을 보유한다.

```csharp
public const int InitialHandSize = 2;  // Stage 시작 시 드로우 장수
public int MaxHandSize => 3 + Stats.CardCap;  // 핸드 최대 장수 (장비 StatCardCap 보너스 포함)
public const int DrawPerTurn = 1;       // 턴당 드로우 장수

public void BuildCardPool()  // 장착 장비 기반 카드풀 생성 + 셔플
public void InitHand()        // 초기 핸드 드로우 (InitialHandSize)
public void DrawCards(int count)  // 드로우, 풀 고갈 시 소모풀 재충전
public void UsePawnCard(DCard card)  // 카드 사용, Supply/Consume은 소멸, 나머지 소모풀로
```

- **카드풀 생성**: `StageScript.OnEnterScene`에서 배치 후 `pawn.BuildCardPool()` + `pawn.InitHand()` 호출
- **턴 드로우**: `TurnManager.StartPawnTurn`에서 `pawn.DrawCards(DPawn.DrawPerTurn)` 호출
- **카드 사용**: `StageManager.UseCardSelf/TryUseCard`에서 `cardOwner.UsePawnCard(card)` 호출
- **소모풀 재충전**: `_pawnDiscardPool` → `_pawnCardPool` 셔플 (자동)
- 장비 미장착 Pawn은 카드풀이 비어 핸드 0장

## Pawn 배치 시스템

스테이지 진입 시 `DeckManager.DeckPawns`의 Pawn을 `TileMapPreset`의 Player 스폰 포인트에 자동 배치한다.

### TileMapPreset 배치 관련 필드

| 필드 | 타입 | 설명 |
|------|------|------|
| `turnLimit` | int | 턴 제한 (0 = 무제한) |
| `maxPawnCount` | int | 배치 가능한 최대 Pawn 수 (0 = 스폰 포인트 수만큼) |
| `tiles[i].spawnPoint` | SpawnPointType | Player / Enemy / None |
| `tiles[i].spawnId` | int | Enemy 전용 Monster ID (Player 스폰에서 무시) |

### 배치 수 결정 공식

```
배치 수 = min(maxPawnCount, 스폰 포인트 수, DeckManager.DeckPawns.Count)
```

### 배치 흐름

```
StageScript.OnEnterScene()
    └── StageManager.LoadStage(preset)
            └── LoadStageFromPreset(preset)
                    1. TileManager.GetSpawnPoints(Player) → 배치 위치 목록
                    2. DeckManager.DeckPawns → 배치할 Pawn 목록
                    3. count = min(pawnLimit, spawnPoints, deckPawns)
                    4. for i in count:
                            Resources.Load(pawn.Data.PrefabPath)
                            Instantiate → Actor.Init(pawn)
                            tile.Slot.SetEntity(actor)
                            _dicUserActor.Add(i, actor)
```

### DeckManager 관련 API

```csharp
public IReadOnlyList<DPawn> DeckPawns => _deckPawnList;  // 덱 Pawn 목록
public void AddPawn(DPawn pawn)                           // 영입된 Pawn 추가
```
